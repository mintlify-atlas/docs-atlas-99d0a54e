These endpoints return orders scoped strictly to the authenticated user. Authentication via a valid session cookie is required for both routes — unauthenticated requests are rejected with `401`. Orders created as guest checkouts (with `userId: null`) are not accessible through this API; only orders linked to an authenticated account are returned.

---

## List orders

Returns all orders belonging to the authenticated user, sorted by `createdAt` descending (most recent first). Each order includes its line items and payment records.

```bash
curl https://api.avanzarintimeshop.com/api/v1/orders \
  -H "Cookie: better-auth.session_token=<your-session-token>"
```

**Requires:** valid session cookie (`requireSession` middleware).

### Response

<ResponseField name="orders" type="Order[]" required>
  All orders belonging to the authenticated user, ordered newest-first.

  <Expandable title="Order object">
    <ResponseField name="id" type="string (UUID)">
      Unique order identifier.
    </ResponseField>
    <ResponseField name="orderNumber" type="string">
      Human-readable order reference (e.g., `"ORD-20241115-A3F2"`). Unique across all orders.
    </ResponseField>
    <ResponseField name="userId" type="string">
      The authenticated user's ID. Always present on orders returned by this endpoint.
    </ResponseField>
    <ResponseField name="status" type="string (enum)">
      Current order status. One of: `pending_payment`, `paid`, `preparing`, `shipped`, `delivered`, `cancelled`.
    </ResponseField>
    <ResponseField name="currency" type="string">
      ISO 4217 currency code for all monetary values in this order.
    </ResponseField>
    <ResponseField name="buyerName" type="string">
      Snapshot of the buyer's name at time of order.
    </ResponseField>
    <ResponseField name="buyerEmail" type="string">
      Snapshot of the buyer's email at time of order.
    </ResponseField>
    <ResponseField name="buyerPhone" type="string">
      Snapshot of the buyer's phone number at time of order.
    </ResponseField>
    <ResponseField name="shipRecipient" type="string">
      Snapshot of the recipient's name at time of order.
    </ResponseField>
    <ResponseField name="shipProvince" type="string">
      Destination province, snapshotted at time of order.
    </ResponseField>
    <ResponseField name="shipMunicipality" type="string">
      Destination municipality, snapshotted at time of order.
    </ResponseField>
    <ResponseField name="shipAddressLine" type="string">
      Full delivery address, snapshotted at time of order.
    </ResponseField>
    <ResponseField name="subtotalMinor" type="integer">
      Sum of all line totals in cents.
    </ResponseField>
    <ResponseField name="shippingMinor" type="integer">
      Shipping cost in cents.
    </ResponseField>
    <ResponseField name="discountMinor" type="integer">
      Discount amount in cents. `0` if no discount was applied.
    </ResponseField>
    <ResponseField name="totalMinor" type="integer">
      Final order total in cents (`subtotalMinor + shippingMinor - discountMinor`).
    </ResponseField>
    <ResponseField name="createdAt" type="string (ISO 8601)">
      Timestamp when the order was placed.
    </ResponseField>
    <ResponseField name="items" type="OrderItem[]">
      Line items snapshot. Each entry contains `{ id, orderId, productId, productName, unitAmountMinor, quantity, lineTotalMinor }`. `productId` may be `null` if the product was later deleted.
    </ResponseField>
    <ResponseField name="payments" type="Payment[]">
      Payment records for this order. Typically one entry. Each contains `{ id, orderId, method, status, amountMinor, currency, reference, confirmedAt }`.
    </ResponseField>
  </Expandable>
</ResponseField>

## Get a single order

Returns one order by its UUID, but only if it belongs to the authenticated user. If the order exists but belongs to a different user, it returns `404` — not `403` — to avoid leaking existence information.

The single-order response also includes `statusHistory`: a chronological audit trail of every status transition the order has undergone.

```bash
curl https://api.avanzarintimeshop.com/api/v1/orders/ord-0001-0000-0000-0000-000000000001 \
  -H "Cookie: better-auth.session_token=<your-session-token>"
```

**Requires:** valid session cookie (`requireSession` middleware).

### Path parameters

<ParamField path="id" type="string (UUID)" required>
  The unique identifier of the order to retrieve. Only orders belonging to the authenticated user are returned — all others yield `404`.
</ParamField>

### Response

<ResponseField name="order" type="Order" required>
  The matching order with items, payments, and full status history.

  <Expandable title="Additional field on single-order response">
    <ResponseField name="statusHistory" type="OrderStatusHistory[]">
      Chronological audit trail of status changes, ordered by `createdAt` ascending. Each entry contains `{ id, orderId, status, changedBy, createdAt }`. `changedBy` is the admin user ID who made the change, or `null` for system-generated entries (e.g., initial `pending_payment` created at checkout).
    </ResponseField>
  </Expandable>
</ResponseField>
