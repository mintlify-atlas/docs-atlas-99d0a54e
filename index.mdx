Avanzar In Time Shop is a production-ready, self-hosted e-commerce platform built as a Bun monorepo. It ships three tightly integrated apps — a **Hono REST API backend**, a **React admin dashboard**, and an **Astro storefront** — backed by PostgreSQL via Drizzle ORM. The platform is purpose-built for manual payment confirmation workflows (Zelle, bank transfers, cash on delivery) and supports guest and authenticated checkout with server-side price calculation and atomic stock management.

## Key Features

<CardGroup cols={2}>
  <Card title="Multi-Currency Pricing" icon="coins">
    Products support multiple currencies with prices stored in minor units (cents) per ISO 4217.
  </Card>
  <Card title="Manual Payment Confirmation" icon="circle-check">
    Admin staff confirm payments (Zelle, transfer, COD) manually with full audit trail.
  </Card>
  <Card title="Order State Machine" icon="arrow-right-arrow-left">
    Orders follow a strict lifecycle: pending_payment → paid → preparing → shipped → delivered / cancelled.
  </Card>
  <Card title="Guest Checkout" icon="user-check">
    Customers can check out without creating an account. Authenticated users get order history.
  </Card>
  <Card title="Atomic Stock Control" icon="warehouse">
    Stock is decremented atomically at checkout to prevent overselling under concurrency.
  </Card>
  <Card title="Role-Based Access" icon="shield-halved">
    Three roles (customer, staff, admin) with route-level enforcement via Better Auth.
  </Card>
</CardGroup>

## Get Started

<Steps>
  <Step title="Install dependencies">
    Clone the repo and install all workspace dependencies with Bun.
    ```bash
    git clone https://github.com/ItsJhonAlex/Ecommerce.git
    cd Ecommerce
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    Copy `.env.example` to `.env` and fill in your database credentials and Better Auth secret.
    ```bash
    cp .env.example .env
    ```
  </Step>
  <Step title="Start the database">
    Spin up PostgreSQL with Docker Compose, then run migrations.
    ```bash
    bun run db:up
    bun run db:migrate
    ```
  </Step>
  <Step title="Run all apps">
    Start the backend, admin dashboard, and storefront concurrently.
    ```bash
    bun run dev
    ```
  </Step>
</Steps>

<Note>
  The backend runs on port **3000**, the storefront on **4321**, and the admin dashboard on **5174** by default. See the [environment variables reference](/configuration/environment-variables) for all available options.
</Note>
